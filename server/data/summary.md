在这次项目开发过程中，我一边查资料一边调试代码，逐步把这些内容串联起来，也对实际项目开发有了更深的体会。

## 二、主要学习内容

### 1. React 组件化开发

通过这次项目练习，我进一步理解了 **React 组件化开发** 的思想。一个完整的系统并不是一次性写在一个页面中，而是需要根据功能拆分为多个独立模块，例如：

- 登录页
- 工作台
- 课程管理
- 学生管理
- 学习总结

每个模块既有自己的职责，也能够组合成完整系统。

在开发过程中，我体会到组件化最大的优点就是：

- **复用性更强**
- **可维护性更高**
- **代码结构更清晰**

比如表格区域、筛选区域、弹窗表单等功能，都可以按照页面结构分层组织。这样写出来的代码更清晰，后期修改也更方便。

```tsx
function MainLayout() {
  return (
    <Layout>
      <Sidebar />
      <Content />
    </Layout>
  )
}
```

---

### 2. TypeScript 的使用

在使用 React 开发的同时，我也接触并练习了 **TypeScript**。相比只写 JavaScript，TypeScript 增加了类型约束，这一开始会让我觉得写代码更麻烦，但在后面开发中我发现它其实很有帮助。

例如在以下场景中：

- 表单数据
- 接口返回值
- 表格列数据
- 组件 props 定义

如果提前定义好类型，就能减少很多低级错误。尤其是在页面字段比较多的时候，TypeScript 能帮助我更快发现：

- 属性名写错
- 数据类型不匹配
- 参数结构不一致

也让代码结构更加规范。

```ts
interface CourseItem {
  id: number
  name: string
  teacher: string
  status: string
}
```

---

### 3. 路由与页面切换

本项目包含多个业务页面，因此我练习了前端路由的基本使用方法。通过路由配置，不同地址可以对应不同页面组件，用户点击菜单后可以在系统中切换功能模块。

除了普通页面跳转，我还学习了 **路由守卫** 相关的实现思路。例如用户没有登录时，不能直接访问工作台、课程管理等页面，而是应该跳转回登录页。

这个功能让我理解了：

> “页面能打开”和“页面有权限访问”，其实是两件不同的事情。

通过这个过程，我对前端项目中的 **登录状态管理** 和 **权限控制逻辑** 有了更实际的认识。

```tsx
if (!localStorage.getItem('token')) {
  return <Navigate to="/login" replace />
}
```

---

### 4. 接口请求与前后端联调

前端项目不能只停留在静态页面展示，更重要的是和后端接口进行联调。在这次项目中，我练习了使用请求工具向后端发送数据请求，并根据接口返回结果更新页面内容。

例如：

- 登录功能需要把用户名和密码提交到后端验证
- 课程管理和学生管理页面需要从后端获取列表数据
- 新增、编辑、删除操作也都依赖接口完成

通过这些功能的实现，我逐渐理解了前端和后端之间的数据交互过程，也体会到 **接口字段对应关系的重要性**。

---

### 5. Ant Design 组件的使用

在界面开发过程中，我较多使用了 **Ant Design** 提供的现成组件，如：

- 表单
- 输入框
- 按钮
- 表格
- 弹窗
- 分页
- 菜单

使用组件库可以提高开发效率，也能让页面样式更加统一。

不过在使用过程中我也发现，组件库虽然方便，但并不是拿来就能完全符合需求，很多时候还是需要自己调整属性、处理状态、补充交互逻辑。比如：

- 表格分页
- 弹窗回显
- 确认删除提示
- 菜单高亮

这些都需要结合实际需求进行处理。也正是在这个过程中，我对组件的使用不再只是“会调用”，而是开始关注 **组件行为与业务逻辑的配合**。

### 6. 页面样式与布局设计

除了功能实现，我也对页面布局和界面展示进行了练习。例如整体后台管理系统采用了：

- 左侧菜单
- 顶部导航
- 内容区展示

这种布局方式比较适合多模块管理系统。通过这次项目，我对后台系统常见页面结构有了更实际的认识。

在样式方面，我也练习了如何让界面更整洁，比如：

- 控制间距
- 设置圆角
- 调整图标
- 优化颜色搭配
- 让表格和卡片区域区分更明显

虽然我在审美和细节处理上还有提升空间，但相比之前只追求“能显示出来”，现在会更关注页面是否 **清晰**、是否 **易用**。

---

### 7. 数据可视化展示

在工作台模块中，我接触了数据可视化相关内容。课程总数、学生总数、发布率、活跃率通过统计卡片展示，而课程排行、学习活跃度、学生状态分布和课程分类分布则通过图表进行展示。

这个过程让我认识到，前端不仅是展示数据，还要思考如何让数据更容易被理解。不同数据适合不同图表形式，例如：

- **趋势数据** 适合折线图
- **分类占比** 适合饼图
- **对比数据** 适合柱状图

通过这部分开发，我对下面这句话有了更深的理解：

> 可视化不是简单地堆图表，而是更清楚地表达信息。

---

### 8. Markdown 内容渲染

学习总结模块使用了 **Markdown 文件** 存储内容，再由前端解析并渲染展示。这个功能让我了解到，前端页面不一定所有内容都要写死在组件中，也可以通过读取外部内容文件的方式来实现动态展示。

这种实现方式让我对 **“内容和页面结构分离”** 有了更深的认识：

- 总结内容写在 Markdown 文件里
- 页面负责解析和展示内容
- 后续修改内容时更方便
- 也比直接写在页面代码中更清晰

这让我意识到，前端开发不仅要考虑页面怎么写，还要思考 **内容组织方式是否合理**。

## 四、开发过程中遇到的问题

在项目开发过程中，我也遇到了不少问题。

### 1. 路由跳转和登录状态处理不熟悉

刚开始做登录功能时，我只关注了表单提交和页面跳转，没有真正把“登录状态校验”考虑完整。后来在实现未登录跳转时，才逐渐理解应该在页面访问前判断 token 是否存在，否则用户仍然可以直接输入地址进入内部页面。

这一问题让我意识到：

> 登录成功只是第一步，登录状态的持续校验同样重要。

---

### 2. 表格分页和筛选逻辑容易混乱

课程管理和学生管理页面都涉及分页、搜索和筛选。最开始我对这些参数的处理不够清楚，经常会出现：

- 翻页后筛选条件丢失
- 重新搜索后页码没有重置
- 查询条件和分页状态混在一起

后来我逐步理清了 **“查询条件”** 和 **“分页参数”** 之间的关系，页面逻辑才更加稳定。

---

### 3. 新增和编辑表单共用时容易出错

在弹窗表单中，新增和编辑往往共用同一个表单组件。开始时我没有处理好默认值和回显逻辑，导致：

- 编辑时数据不能正常显示
- 关闭弹窗后再次打开会残留上一次的数据
- 表单重置不彻底

通过不断调试，我对 **表单重置、回填和提交逻辑** 有了更清楚的认识。

---

### 4. 前后端联调时字段对应问题较多

前端页面显示的字段名称和后端接口返回的字段不一致时，很容易出问题。比如有时是：

- 字段名不同
- 类型不同
- 接口返回结构和预期不一致

这让我意识到，写前端不能只看页面，也要认真理解接口文档和数据结构。只有把前后端字段关系理顺，页面数据展示才会稳定。

---

### 5. 生产环境部署思路一开始不清楚

我一开始更熟悉开发环境，只知道 `npm run dev` 可以打开页面，但对以下问题理解并不深：

- 打包后怎么访问
- 为什么要用 Koa 托管静态资源
- 为什么开发环境和生产环境访问方式不同

完成这次作业后，我对 **开发代理、前端打包和后端托管静态资源之间的关系** 清楚了很多，也对项目从开发到部署的完整流程有了更实际的认识。

```js
app.use(serve(path.join(__dirname, '../client/dist')))
```
