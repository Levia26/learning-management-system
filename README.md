# 在线学习管理平台（Learning Management System）

一个基于 **React + TypeScript + Ant Design + Tailwind CSS** 构建前端、基于 **Node.js + Koa + SQLite** 构建后端的学习管理平台项目。

该项目以后台管理系统为核心场景，围绕课程管理、学生管理、数据统计分析和学习内容展示等功能展开，完整实现了从登录认证、业务列表管理到数据可视化展示的基础流程。

项目重点体现了前后端联调、管理后台开发、表单与表格交互、接口调用、状态管理以及部署联通等能力。

---

## 📌 项目背景

在管理类系统开发中，课程信息、学生信息以及学习数据的统一管理是较为典型的业务场景。本项目以“在线学习管理平台”为主题，设计并实现了一个包含登录认证、统计看板、课程管理、学生管理、学习总结展示等功能的后台系统。

项目采用前后端分离模式开发：

- 前端：页面搭建、交互逻辑、数据展示、图表可视化
- 后端：接口提供、数据管理、认证处理、静态资源托管

通过该项目，我系统梳理并实践了后台管理系统开发中的常见模块与开发流程。

## 技术实现说明

### 1. 登录认证与路由拦截
项目中通过登录接口获取 token，并将登录态保存在本地。前端在路由层面对未登录访问进行拦截，保证只有认证用户可以进入后台页面。

### 2. 列表页交互设计
课程管理和学生管理页面都采用“查询条件 + 表格 + 分页 + 弹窗表单”的典型后台模式实现。通过统一组织查询参数和表单状态，使页面交互更加清晰。

### 3. 图表数据处理
后端返回的统计数据需要经过前端转换后才能适配图表组件，因此在实现过程中需要根据柱状图、折线图和饼图的不同数据结构分别处理，提高组件复用性和展示稳定性。

### 4. 开发环境代理
由于前端运行在 `5173` 端口、后端运行在 `3000` 端口，开发环境中通过 Vite 配置代理，将 `/api` 请求转发至后端，避免跨域问题并保持统一请求方式。

### 5. 生产环境静态资源托管
前端执行构建后输出到 `client/dist`，后端通过 Koa 托管该目录中的静态资源，从而支持在 `http://localhost:3000` 下直接访问完整项目。

## 开发过程中的重点与难点

### 1. 表格与表单联动较多
后台管理系统中最核心的问题不是单一页面，而是“查询、表格、弹窗、编辑回显、提交刷新”这一整套链路是否顺畅。开发过程中需要不断梳理状态流转，避免新增、编辑、筛选之间互相影响。

### 2. 图表展示依赖数据结构转换
图表本身不难，真正需要注意的是后端返回的数据格式与图表组件要求的数据结构之间的映射关系，尤其是在多图表同时展示时，要保证字段含义与展示逻辑一致。

### 3. 前后端运行环境不同
开发环境与生产环境下的访问方式不同。开发时需要代理，生产时需要静态托管，因此在项目联通时需要分别处理接口路径与资源访问问题。

### 4. 管理系统交互细节较多
例如删除确认、状态切换、列表刷新、分页参数保留、编辑回显等，这些细节虽然单独看不复杂，但组合在一起时很容易影响整体体验，因此在实现时需要反复调整。
